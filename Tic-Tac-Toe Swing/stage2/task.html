Description

In this step, we will add an event listener for the field cells. The first time you click on a free cell, the “X” symbol should appear, while the second click on another free cell, the “O” symbol. We will also add a status bar and a “Reset” button. When you click on the "Reset" button, the playing field must be cleared. These improvements are enough for you to play Tic-Tac-Toe against a friend using the created program.

Objectives

You must add a listener for each cell of the field. When you click on a free cell, the symbol "X" or "O" should appear, depending on whose move. If the user clicks on a cell where there is already a character, such a click should be ignored.

The program should track the status of the game. In the event of the end of the game, further pressing on free cells of the field should be ignored. At any time, when you press the "Reset" button, the field should be cleared and the game starts again.

The status field should contain one of the following messages:

  • The game is not started
  • The game is playing
  • X wins
  • O wins
  • Draw

For testing purposes, you must add names for the following components:

  • "LabelStatus" for the component where the game status is displayed
  • "ButtonReset" for reset button

The names for the cells of the playing field should remain the same as in the previous stage. To display symbols, you must set the button text. It must be one of three strings: “ “ for an empty cell, “X” for a first player and “O” for a second player.

To remove the border around the text of the active cell, use the method setFocusPainted(false) when creating the field cell component.

You can set any font, size and color for the cell symbols and any background for the cell itself. The tests only check the names of the components and their text.
