Description

In this step, we will teach the computer to play tic-tac-toe. We will add a control panel to our program where you can choose who will play for noughts and crosses. We will also add a start button to start the game.

Objectives

You must develop and implement a tic-tac-toe strategy for the computer. The strategy can be as complex, for example as minimax, or the simplest one, where the computer chooses a random free field. You are given complete freedom in choosing a strategy.

A toolbar with three buttons should be added to the program. These are buttons for selecting the first and second player and starting and resetting the game. When you click on the player's button, the program should switch from a person to a robot and vice versa. The program should support four game modes: human versus human, human versus robot, robot versus human and robot versus robot.

Optionally, you can add a small delay (about a second) when the robot plays. Thus, if you choose to play a robot against a robot, you will be able to observe the process of the game.

Due to testing reasons, you need to name some components.

Set the names to these components:

  • Nine  components for cells from "ButtonA1" to "ButtonC3"
  • Status bar component to "LabelStatus"
  • Player choosers to "ButtonPlayer1" and "ButtonPlayer2"
  • Start/Reset button to "ButtonStartReset"
